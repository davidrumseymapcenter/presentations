historical maps as data

Traditionally, historical maps have been utilized as visual geographic aids that align with specific research objectives. Researchers might handpick particular maps for detailed examination and analysis related to their work. More recently, as many of you know, historical maps have been brought into a digital world and used with GIS software for spatial analysis. There have been attempts to extract vector data from maps, mostly manually or semi-automatically. Any attempt that I'm aware of to automatically extract vector data has been largely innaccurate and have typically required a lot of time consuming manual clean up. I'm going to mainly focus on map text as data, as opposed to features like roads, rivers, or buildings. The concept of map text as a substantial body of data has remained underexplored—a void that the advent of machines capable of reading and interpreting maps has aimed to fill.
the reason we think maps are beautiful is the same reason we've had trouble with extracting text from them in a programmatic way. Cartographic conventions encourage the creative use of font styles, sizes, orientation, color, for example. But advancements in object detection techniques have made it possible to identify variable text on maps. This is exciting because we've never had the ability to work with map text as a corpus of data before, like Google Books, Internet Archives, other large-scale OCR projects have for standard printed text.

Machines Reading Maps

That's where Machines Reading Maps came in. The Machines Reading Maps project was a grant funded project that aimed to normalize map text as a new kind of data and transform how scholars interpret and interact with this new kind of map data.

MRM's MapKurator System

https://knowledge-computing.github.io/mapkurator-doc
Their flagship system was the mapKurator pipeline, which processes scanned historical map images to spit out text labels and their bounding polygons, and a geo-entity identifier from OpenStreetMap. The full system is made up of several modules to process a large number of high-res map images. ImageCropping breaks down large map images into smaller manageable patches which are then analyzed by PatchTextSpotter for text recognition. This module automatically identifies and interprets the text present on these patches using models trained with both synthetic text data and human-annotated information from the historical document annotation platform Recogito. Then, PatchtoMapMerging involves merging individual patch findings back into a cohesive map view, which enables visualization in tools like QGIS by converting image files back into JPEG. PostOCR then refines the recognized text, cross-referencing words against OpenStreetMap’s extensive toponym dictionary and corrects any misidentifications and improve accuracy. After verification, GeocoordinateConverter translates the identified text locations from pixel coordinates into geographical coordinates. Finally, EntityLinker links detected map texts to potential entities within OpenStreetMap by checking if the recognized text is part of an entity’s name and then confirming that it falls within the map boundaries using data derived from the GeocoordConverter.
The PatchTextSpotter was trained with a few datasets using the TESTER framework - which is transformer-based text detection framework. I mentioned from Recogito human annotations, and two sythnetic datasets: SynthText data and SynMap. Synmap uses OSM basemaps and label text that have been styled like historical maps, and SythnText places words in natural scenes, taking into account scene layout. This is a synthtext dataset.

Transformative Impact of AI on Historical Map Analysis

Now that we've looked at that, I wanted to take a quick step back and talk about the transformative impact of AI on historical map discovery and analysis. Lrge language models have the potential to revolutionize how we explore historical documents, similar to modern generative AI applications we use today, like OpenAI's ChatGPT. These models digest and comprehend enormous amounts of data in one go, offering rich contextual answers, moving beyond traditional query methods like searching text to match keywords found in metadata. But the potential goes beyond just searching for maps and content when we semantically enrich these models with external domain-specific or other external knowledge bases. For example A future step in this technology would be for a user to be able to search for “shipwreck” and finding not only occurrences of the same word in different languages, but also variants and synonyms with closely related meanings. By integrating textual information derived from historical maps with these models, we open the door to a wide range of new possibilities. Domain-specific versions of LLMs could be fine-tuned with cartographic terminology, cataloging metadata, toponyms, and map history literature. Leading to new types of historical geographic analyses. Data quality challenges It’s crucial to remember that maps are never representations of fact; they always carry inherent biases and subjectivity. Still, the development of new tools and approaches with text from historical maps can provide dynamic learning experiences and a deeper understanding of concepts that began as questions. The transparency of its flaws and open-ness of extracted text data allows us to tinker and experiment with its usefulness and understand its full potential.
